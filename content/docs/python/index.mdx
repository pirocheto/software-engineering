## Présentation

Python est un langage de programmation interprété, portable, dynamique, extensible et open-source.
Il permet une approche modulaire et orientée objet sans l'imposer.
Conçu en 1989 par Guido van Rossum, il est aujourd'hui maintenu par la Python Software Foundation.

## Caractéristiques

### Points forts

Python présente de nombreux avantages qui expliquent sa popularité croissante :

- **Interprété** : Pas besoin de compilation, Python exécute directement le code via un interpréteur.
- **Multi-paradigme** : Il prend en charge la programmation impérative, fonctionnelle et orientée objet.
- **Multi-plateformes** : Compatible avec Windows, macOS et Linux, il permet une exécution uniforme sur différents systèmes.
- **Extensible** : Intégration possible avec C, C++ et d'autres langages pour optimiser les performances.
- **Lisible et simple** : Une syntaxe claire et intuitive facilitant l'apprentissage et la maintenance du code.
- **Large écosystème** : Une communauté active et un vaste choix de bibliothèques pour de nombreux domaines (science des données, IA, web, etc.).

### Limites

Malgré ses qualités, Python présente quelques inconvénients :

- **Performances moindres** : Plus lent que les langages compilés comme C ou Rust.
  Rassurez-vous, la plupart des applications n'ont pas besoin de performances extrêmes.
  Vous convrirez la plupart de vos besoins en suivant ces recommandations :

  1. Concentrez-vous d'abord sur l'optimisation de votre code en respectant les bonnes pratiques de programmation.
  2. Utilisez des bibliothèques optimisées (ex : NumPy pour le calcul scientifique).
  3. Pour des cas spécifiques, recourez à des extensions en C ou Rust pour les parties critiques. (compétences avancées)

- **Gestion de la mémoire** : L'utilisation d'un ramasse-miettes (garbage collector) peut ralentir les applications gourmandes en mémoire.

- **Multi-threading limité** : Le GIL (Global Interpreter Lock) restreint l'exécution parallèle des threads.
  Solutions possibles :
  1. Utilisation du module multiprocessing.
  2. Recours à des frameworks comme Dask ou Ray pour le parallélisme distribué.

Malgré ces limites, sachez que l'écosystème Python est si riche en bibliothèques et outils qu'il est preque toujours possible de trouver une solution à vos problèmes.

## Domaines d'application

Python est utilisé dans divers domaines, avec des outils et bibliothèques adaptés :

- **Automatisation et scripting** : Idéal pour l'automatisation des tâches répétitives et la gestion des fichiers.
- **Développement web** : Frameworks populaires comme Django, Flask et FastAPI.
- **Analyse de données** : Manipulation et visualisation avec Pandas, NumPy et Matplotlib.
- **Intelligence artificielle** : Apprentissage automatique avec Scikit-learn, TensorFlow et PyTorch.
  Langchain et LLamaindex pour les LLM (Large Language Models).
- **Cybersécurité** : Utilisé pour l'analyse de logs, l'automatisation des tests de sécurité.
- **Systèmes embarqués** : Compatible avec Raspberry Pi et d'autres microcontrôleurs.

Python est moins recommandé pour :

- **Développement de jeux** : Bien que possible avec Pygame, d'autres langages comme C# (Unity) ou C++ (Unreal Engine) sont préférables.
- **Applications mobiles** : Des frameworks comme Kivy existent, mais Java/Kotlin (Android) et Swift (iOS) restent dominants.
- **Interfaces graphiques complexes** : PyQt et Tkinter existent, mais JavaScript et ses frameworks sont plus appropriés.

## Un peu d'histoire

### Origines

Python a vu le jour à la fin des années 1980 sous l'impulsion de Guido van Rossum, alors chercheur au CWI (Centrum Wiskunde & Informatica) aux Pays-Bas. Inspiré par le langage ABC, il voulait créer un langage simple et puissant.

### Évolutions majeures [!toc]

#### Python 1.0 (1991) [!toc]

- Types de données de base (listes, dictionnaires, etc.).
- Gestion des exceptions et modules intégrés.

#### Python 2.0 (2000) [!toc]

- Introduction du garbage collector.
- List comprehensions.
- Support partiel de l'Unicode.

Longue transition vers Python 3 due à des incompatibilités.

#### Python 3.0 (2008) [!toc]

- Réécriture majeure du langage.
- Amélioration de la gestion de l'Unicode.
- Suppression des anciennes syntaxes obsolètes.
- Fin du support de Python 2 (2020)
- Adoption généralisée de Python 3.

### Python aujourd'hui

Python est aujourd'hui l'un des langages les plus utilisés au monde, soutenu par une communauté dynamique et un écosystème riche. Sa flexibilité en fait un choix privilégié pour le développement logiciel, l'analyse de données et l'intelligence artificielle.
